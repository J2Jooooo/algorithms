Computer Systems: A Programmer's Perspective《深入理解计算机系统》

>如果你完全理解底层计算机系统以及它对应用程序的影响，那么你将成为凤毛麟角的'权威'程序员”

   
# 第一阶段：
第1章 计算机系统漫游  18页   一周  
第一周先慢启动。

第2章  信息的表示和处理   60页  两周  
这一章很枯燥，关键是理解整数和浮点数是怎么表示的。  
可以跳过数学公式。

第3章  程序的机器级表示    100页   3周  
程序在机器级别是怎么运算的，重点理解函数调用在机器级别怎么实现。

# 第二阶段
第4章  处理器体系结构  
纯硬件，可以跳过

第5章  优化程序性能  
和硬件密切相关，感兴趣的可以阅读

第6章 存储器层次结构  ~30页 一周  
除去硬件部分，大概30页左右  一周可以搞定

第7章 链接  ~26页，时间待定  
非常难的一章。 

第8章 异常控制流   ~45页   2周时间  
主要讲系统中断和系统调用：fork,wait,execve 信号

第9章 虚拟存储器   ~50页 ， 时间待定  
主要讲页表，地址翻译， 动态存储器的分配（malloc）

# 第三阶段
第10章 系统级IO  ~16页，时间待定  
C语言级别的文件读写，重定向

第11章 网络编程  ~30页  
基本的网络知识，socket,http  
用socket实现Web服务器

第12章 并发编程   ~40页  
多进程,多路复用,多线程，信号量，生产者消费者问题，竞争和死锁
